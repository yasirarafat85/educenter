# EduCenter — সম্পূর্ণ সাইট সেটআপ গাইড (cPanel শেয়ার হোস্টিং)

## সাইটের সংক্ষিপ্ত পরিচিতি
- **স্ট্যাক:** Native PHP + MySQL (কোনো framework/Composer/SSH লাগে না, শেয়ার হোস্টিং এ সরাসরি চলে)
- **পাবলিক সাইট:** হোম, কোর্স, ওয়ার্কশিট, প্রোডাক্ট, ডিটেইল পেজ, নোটিশ, শিক্ষক, রিভিউ, অ্যাবাউট, গ্যালারি, FAQ — সব ডাটাবেস থেকে dynamic
- **অ্যাডমিন প্যানেল:** `/admin/` — সব কনটেন্ট, সাইট সেটিংস, রেজিস্ট্রেশন/অর্ডার ম্যানেজ করা যায়, কোড টাচ ছাড়াই
- **রেজিস্ট্রেশন/অর্ডার ফর্ম:** ওয়ার্কশিট/প্রোডাক্টের জন্য সাধারণ ফর্ম (নাম/ফোন/ঠিকানা/জেলা-থানা)। কোর্সের জন্য আলাদা ২-ধাপের কালারফুল ফর্ম (`course-register.php`) — আগে কোর্স নির্বাচন, তারপর শিশুর নাম/জন্ম তারিখ/ফেসবুক আইডি/বাবা-মায়ের মোবাইল/রিসিভার তথ্য সহ ফর্ম, মোবাইল নম্বর দিয়ে আগের তথ্য অটো-ফিল হয়
- **কুরিয়ার API:** Steadfast Courier ইন্টিগ্রেশন রেডি (অন্য প্রোভাইডার পরে যোগ করা যাবে)
- **আয়-ব্যয়:** রেজিস্ট্রেশন/অর্ডার "আয় হিসেবে অনুমোদন" করলে অটো আয় যোগ হয় (কোর্স/ওয়ার্কশিট/প্রোডাক্ট ক্যাটেগরি অনুযায়ী), ম্যানুয়াল আয়/খরচও যোগ করা যায় নতুন কাস্টম ক্যাটেগরি দিয়ে
- **নিরাপত্তা:** ব্রুট-ফোর্স প্রতিরোধ, CSRF প্রোটেকশন, session hardening, security headers, HTTPS enforcement (নিচে বিস্তারিত)

## ধাপ ১ — MySQL ডাটাবেস বানান
1. cPanel এ লগইন করুন
2. **MySQL® Databases** এ যান
3. একটা নতুন Database বানান (যেমন: `main`)
4. একটা নতুন Database User বানান, শক্তিশালী পাসওয়ার্ড দিন
5. সেই User কে Database এ **All Privileges** দিয়ে Add করুন
6. Database ও User এর পুরো নাম নোট করে রাখুন (cPanel সাধারণত সামনে `cpaneluser_` prefix জুড়ে দেয়, যেমন `educ1234_main`)

## ধাপ ২ — Schema Import করুন
1. cPanel এ **phpMyAdmin** খুলুন
2. বাম পাশ থেকে আপনার নতুন ডাটাবেস সিলেক্ট করুন
3. উপরে **Import** ট্যাবে ক্লিক করুন
4. `database/schema.sql` ফাইলটি বেছে নিন এবং নিচে **Go** চাপুন
5. সব টেবিল তৈরি হয়ে গেলে সবুজ success মেসেজ দেখাবে — মোট ১৯টা টেবিল তৈরি হবে (courses, worksheets, products, teachers, reviews, notices, gallery, faqs, settings, admin_users, login_attempts, phone_lookup_attempts, registrations, courier_shipments, finance_categories, income, expenses সহ)

## ধাপ ৩ — ফাইল আপলোড করুন
`D:\clude_project\educenter-deploy.zip` (হালনাগাদ, security fix সহ) ফাইলটি cPanel **File Manager** দিয়ে `public_html/` এ আপলোড করে **Extract** করুন — যাতে `public_html/index.php` হয়, `public_html/website/index.php` না। অথবা FTP দিয়ে পুরো ফোল্ডারের সব ফাইল/সাবফোল্ডার আপলোড করুন।

## ধাপ ৪ — config.php আপডেট করুন
`config.php` ফাইলটি এডিট করে এই মানগুলো বসান:
```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'আপনার_ডাটাবেস_নাম');
define('DB_USER', 'আপনার_ডাটাবেস_ইউজার');
define('DB_PASS', 'আপনার_ডাটাবেস_পাসওয়ার্ড');
define('SITE_URL', 'https://yourdomain.com');
define('DEV_MODE', false);   // লাইভে গেলে অবশ্যই false — নাহলে এরর মেসেজে ভেতরের তথ্য দেখা যাবে
```
`DEV_MODE` false করলে সাইট নিজে থেকেই http:// থেকে https:// তে রিডাইরেক্ট করবে (SSL চালু থাকলে)।

## ধাপ ৫ — যাচাই করুন

## ধাপ ৭ — সেফটির জন্য ডিলিট করুন
এডমিন অ্যাকাউন্ট বানানোর পর নিচের দুইটা ফাইল সার্ভার থেকে ডিলিট করে দিন (এগুলো শুধু সেটআপের জন্য, খোলা রাখলে অন্য কেউ অ্যাক্সেস করতে পারবে):
- `test-connection.php`

## ধাপ ৮ — এডমিন প্যানেল ব্যবহার
`yourdomain.com/admin/login.php` এ গিয়ে লগইন করুন। এখান থেকে যা করা যাবে:
- **কোর্স, ওয়ার্কশিট, প্রোডাক্ট, শিক্ষক, রিভিউ, নোটিশ, গ্যালারি, FAQ** — প্রতিটার জন্য আলাদা মেনু আছে, যোগ/এডিট/ডিলিট সব করা যাবে (ছবি আপলোড সহ)
- **সাইট সেটিংস** — সাইটের নাম, লোগো, যোগাযোগের তথ্য, সোশ্যাল মিডিয়া লিংক, এবং কুরিয়ার API কী এখান থেকে পরিবর্তন হয়
- **রেজিস্ট্রেশন/অর্ডার** — কাস্টমাররা ফর্ম পূরণ করলে এখানে দেখা যাবে, স্ট্যাটাস পরিবর্তন করা যাবে (পেন্ডিং → কনফার্ম → শিপড → ডেলিভার্ড)
- **পাসওয়ার্ড পরিবর্তন** — নিজের এডমিন পাসওয়ার্ড বদলানোর জন্য

## ধাপ ৯ — কুরিয়ার API চালু করা (ঐচ্ছিক)
বর্তমানে **Steadfast Courier** এর জন্য ইন্টিগ্রেশন রেডি করা আছে (উদাহরণ হিসেবে, কারণ এটা সহজ ও জনপ্রিয়)। চালু করতে:
1. Steadfast এর মার্চেন্ট প্যানেল থেকে API Key ও Secret Key সংগ্রহ করুন
2. Admin Panel → **সাইট সেটিংস** → "কুরিয়ার API" সেকশনে গিয়ে Provider এ `steadfast` লিখুন এবং Key/Secret বসান
3. এরপর যেকোনো রেজিস্ট্রেশন/অর্ডারের বিস্তারিত পেজে "📦 কুরিয়ারে পাঠান" বাটন দেখা যাবে

অন্য কুরিয়ার (Pathao/RedX) ব্যবহার করতে চাইলে জানাবেন — `includes/courier/` ফোল্ডারে একই প্যাটার্নে নতুন প্রোভাইডার ক্লাস যোগ করে দেওয়া যাবে।

## ধাপ ১০ — কোর্স রেজিস্ট্রেশন ফর্ম ও "পার্সেল হাইড"
- সাইটে যেকোনো "Register" বাটনে ক্লিক করলে প্রথমে কোর্স তালিকা দেখায় (`course-register.php`), তারপর কোর্স বেছে নিলে বিস্তারিত ফর্ম আসে
- অ্যাডমিন প্যানেল → **কোর্স সমূহ** → কোনো কোর্স এডিট করলে **"পার্সেল হাইড"** নামে একটা চেকবক্স আছে — এটা চালু (Yes) করলে সেই কোর্সের রেজিস্ট্রেশন ফর্মে রিসিভার নাম/নম্বর/ঠিকানা হাইড হয়ে যাবে (যেসব কোর্স সম্পূর্ণ অনলাইনে হয়, পার্সেল/বই পাঠানোর দরকার নেই, সেগুলোর জন্য চালু করবেন)
- মায়ের মোবাইল নম্বর ঘরে আগে থেকে রেজিস্ট্রেশন থাকলে বাকি তথ্য অটোমেটিক ভরে যাবে:
  - বাবার মোবাইল/রিসিভার নাম/নম্বর/ঠিকানা (পরিবার-পর্যায়ের তথ্য) — সবসময় সরাসরি ভরে যায়
  - শুধু ১টা শিশুর রেকর্ড থাকলে তার নাম/জন্ম তারিখ/ফেসবুক আইডিও সরাসরি ভরে যাবে
  - **একাধিক ভিন্ন শিশু** থাকলে (যেমন একই মায়ের ২য়/৩য় সন্তান) চুপচাপ ভুল শিশুর তথ্য না বসিয়ে একটা ছোট সিলেক্টর দেখাবে — "এই নম্বরে আগে নিবন্ধিত শিশুরা: [নাম ১] [নাম ২] [➕ নতুন শিশু]" — সঠিক শিশু বেছে নিলে তার তথ্য ভরবে, "নতুন শিশু" চাপলে নতুন করে লেখা যাবে

## ধাপ ১১ — আয়-ব্যয় (Income & Expenses)
Admin Panel → **আয়-ব্যয়** সেকশনে তিনটা পেজ:
- **ড্যাশবোর্ড** (`finance.php`) — মোট আয়, মোট খরচ, নীট মুনাফা, ক্যাটেগরি অনুযায়ী ব্রেকডাউন
- **আয়** (`income.php`) — ম্যানুয়ালি আয় যোগ করা যায় (ক্যাটেগরি + পরিমাণ + বিবরণ + তারিখ), নতুন ক্যাটেগরি টাইপ করলেই তৈরি হয়ে যায়
- **খরচ** (`expenses.php`) — একইভাবে ম্যানুয়াল খরচ যোগ করার জায়গা, নতুন ক্যাটেগরি (যেমন ভাড়া, বেতন, মার্কেটিং) নিজে থেকে যোগ করা যাবে

**অর্ডার থেকে আয় অটো যোগ হওয়া:**
1. **রেজিস্ট্রেশন/অর্ডার** → কোনো একটা অর্ডার বিস্তারিত পেজে ঢুকুন
2. নিচে **"💰 এই অর্ডারটি আয় হিসেবে অনুমোদন করুন"** ফর্ম দেখবেন — কোর্স/ওয়ার্কশিট/প্রোডাক্টের দাম অনুযায়ী পরিমাণ ও ক্যাটেগরি আগে থেকেই ভরা থাকবে (চাইলে বদলাতে পারবেন, যেমন ছাড় দিলে)
3. **"✅ অনুমোদন করুন"** চাপলে সেই পরিমাণ আয়ের তালিকায় যোগ হয়ে যাবে এবং ড্যাশবোর্ডে দেখা যাবে
4. ভুল হলে **"বাতিল করুন"** দিয়ে অনুমোদন উইথড্র করা যায় (আয়ের এন্ট্রিও মুছে যাবে)

## নিরাপত্তা ফিচার (ইতিমধ্যে চালু আছে)
- **ব্রুট-ফোর্স প্রতিরোধ:** একই IP থেকে ১৫ মিনিটে ৫ বারের বেশি ভুল লগইন চেষ্টা হলে সাময়িকভাবে ব্লক হয়ে যায় (`login_attempts` টেবিল)
- **CSRF প্রোটেকশন:** সব ফর্মে (লগইন, কনটেন্ট এডিট, রেজিস্ট্রেশন) টোকেন যাচাই হয়
- **Session hardening:** কুকিতে HttpOnly + SameSite সবসময়, HTTPS এ থাকলে Secure ফ্ল্যাগও যোগ হয়
- **Security headers:** X-Frame-Options, X-Content-Type-Options, Referrer-Policy
- **সোর্স ফোল্ডার ব্লক:** `includes/`, `admin/includes/` সরাসরি ব্রাউজার দিয়ে অ্যাক্সেস করা যায় না
- **পাসওয়ার্ড হ্যাশিং:** bcrypt (`password_hash`)
- **SQL Injection প্রতিরোধ:** সব ডাটাবেস কোয়েরি prepared statements (PDO) দিয়ে
- **ফাইল আপলোড নিরাপত্তা:** MIME-type যাচাই, random ফাইলনেম, uploads/ ফোল্ডারে PHP এক্সিকিউশন ব্লক
- **HTTPS enforcement:** `DEV_MODE=false` করলে http:// থেকে https:// তে অটো রিডাইরেক্ট
- **ফোন লুকআপ রেট-লিমিট:** কোর্স রেজিস্ট্রেশন ফর্মে মোবাইল নম্বর দিয়ে আগের তথ্য খোঁজার সুবিধাটা প্রতি IP থেকে ১০ মিনিটে সর্বোচ্চ ২০ বার ব্যবহার করা যাবে (`phone_lookup_attempts`) — কেউ যেন ফোন নম্বর ঘুরিয়ে ঘুরিয়ে অন্যদের ব্যক্তিগত তথ্য (শিশুর নাম, ঠিকানা) স্ক্র্যাপ করতে না পারে

**যা আপনাকে করতে হবে:**
- এডমিন পাসওয়ার্ড শক্তিশালী রাখুন (কমপক্ষে ৮-১০ ক্যারেক্টার, সহজে অনুমানযোগ্য না)
- হোস্টিং এ SSL/HTTPS চালু আছে কিনা নিশ্চিত করুন (cPanel → SSL/TLS Status → AutoSSL)
- মাঝে মাঝে phpMyAdmin থেকে ডাটাবেস ব্যাকআপ (Export) নিয়ে রাখুন

## লোকাল XAMPP এ টেস্ট করা
এই প্রজেক্ট `D:\clude_project\website` এ আছে এবং `D:\xampp\htdocs\website` একটা junction দিয়ে সরাসরি এই ফোল্ডারকে পয়েন্ট করে — তাই `http://localhost/website/` এ গেলে সবসময় সর্বশেষ কোড দেখা যাবে, কোনো কপি করার দরকার নেই। লোকাল ডাটাবেসের নাম `educenter` (config.php তে বসানো আছে)।
